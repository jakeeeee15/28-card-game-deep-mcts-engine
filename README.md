# Deep MCTS Twenty-Eight Solver

An advanced, high-performance artificial intelligence agent designed to master the trick-taking card game **28**. This framework implements a hybrid architecture combining deep convolutional/dense policy networks with an asynchronous Monte Carlo Tree Search (MCTS) lookahead engine. By modeling the match as an **Imperfect Information Extensive Form Game**, the solver successfully balances instant policy intuition with deep tactical simulation to eliminate critical blunders.

---

## 🔬 Architecture & Mathematical Foundations

The core decision-making pipeline decouples instinct from raw calculation through a two-stage evaluation layer:

### 1. Intuition Layer (Deep Neural Network Policy)
The agent leverages deep neural network weights (validated at `checkpoint_epoch_82000.pth`) optimized via reinforcement learning self-play. 
* **State Space Encoding:** The raw game layout (hands, history, bidding margins, trump suit status) is flattened into a vectorized tensor.
* **Policy Output:** The network outputs a raw logit distribution over a 32-element action space representing the physical cards in the 28 deck.

### 2. Calculation Layer (Monte Carlo Tree Search)
To protect high-value assets (such as out-of-suit Jacks) from instinctual blunders, the agent passes the state to an explicit MCTS planner. At each turn, time is frozen, and the active `GameState` is deeply cloned into an isolated sandbox to run $N = 150$ simulation trajectories across a 4-phase tree loop:

* **Selection:** The tree is traversed from the root node by choosing actions that maximize the Upper Confidence Bound applied to Trees (UCT) formula:

$$UCT = \frac{W_i}{N_i} + c \times \sqrt{\frac{\ln N_p}{N_i}}$$

Where:
* $W_i$ = Cumulative reward (wins) tracked along branch $i$.
* $N_i$ = Total visit count of the child node $i$.
* $N_p$ = Total visit count of the parent node.
* $c$ = Exploration exploration constant ($\approx 1.41$), balancing known high-yield variations against unvisited branches.

* **Expansion:** When the search reaches a leaf node where the legal action space (generated by the native action-masking matrix) is not fully represented, a new child node is initialized for an unexplored card index.
* **Simulation (Rollout Guided by 82k Weights):** From the expanded node, a stochastic "ghost game" is executed to terminal state. Instead of uniform random choices, moves are sampled using a categorical probability distribution generated by the 82k weights model, heavily improving rollout realism.
* **Backpropagation:** Upon game termination, the scoreboard is evaluated against the contract parameters (`point_call`). If Team A meets or breaks the bid, a reward metric of $R = 1.0$ is propagated back up the path nodes; otherwise, a loss $R = 0.0$ is assigned.

### Strategic Action Optimization
The engine utilizes a deterministic action mask inside `Player.get_the_possible_actions()` to catch edge cases unique to 28's ruleset:
1. **Lead Suit Enforcement:** Restricts the searchable action space if the player can follow the active trick's lead suit.
2. **Trump Reveal Contingency (Action 32):** Explicitly handles the action branch where an asset-starved player requests to lift and reveal the hidden trump card, preventing execution deadlocks in the simulation thread.
3. **Trump Caller Restraints:** Restricts the trump-caller from leading with the trump suit unless forced by a short-suited hand constraint.
